Aspen Frost Visual Brand Standard

Status: Approved visual standard 2026-06-19. Source of truth: design-system/source/aspen-frost-approved-homepage-2026-06-19.html.

This file decomposes the approved Aspen Frost homepage/design-system bundle into rules future website and official branded-content work must follow. The copy in the source artifact is placeholder unless separately approved. The visual system is approved.

Core Idea

Aspen Frost should feel like a working agentic software shop made visible: paper, notebook, dispatch desk, corkboard, trace log, terminal, handoff card, stamp, sticky note, system diagram. It is not a glossy SaaS theme and not a soft studio brand.

The brand tension is human judgment plus agent execution:

  • Human / Andy / approval / judgment: deep spruce and moss green.
  • Agents / queues / traces / execution: cyan frost, not indigo or purple.
  • System work surface: ecru paper with tactile grain, ink, graphite panels, cork, kraft, stamps, and ruled lines.

Mark

Use the six-pointed hub-and-spoke mark from the approved file. It is six cyan agent nodes connected to a central deep-green human hub, with an ecru void at the center.

Official extracted SVG: design-system/assets/aspen-frost-mark.svg.

Rules:

  • Preserve the six-node radial geometry.
  • Preserve the cyan connections/nodes and green center.
  • The mark may sit on ecru paper or be reversed only if the same color relationship remains legible.
  • Do not replace it with the older soft leaf mark or with a generic snowflake/network icon.

Color

Use design-system/tokens.css for implementation. The main tokens are:

  • Paper: #EFE8D6, #E6DDC6, #F6F1E3, #DCD2B8, #C9BD9C.
  • Ink: #232019, #4B463B, #756E5D, #9A917C.
  • Human green: #1E4233, #2E6B4F, #3C8662, #79A088, #DFE7DB.
  • Agent cyan: #2A8AAC, #145468, #54B4CF, #7FD6E4, #DBE7E9.
  • Graphite: #2C2A24, #3B382F, #4E4B40.
  • Tactile accents: cork #BE9258, kraft #C7A06A, post-it #E9D98A, stamp/pin #A8472E, amber #B8791F.

Avoid dominant purple, blue-purple gradients, glossy black, sterile white, and one-note beige. The ecru field is the desk; the green/cyan relationship carries the brand.

Type

Approved families:

  • Display: Newsreader, then Georgia/Times fallback.
  • UI/body: IBM Plex Sans, then system sans fallback.
  • Logs/stamps/metadata: IBM Plex Mono, then system mono fallback.

Use editorial display type for major section statements, Plex Sans for clear working UI, and Plex Mono for stamps, traces, IDs, metadata, and status labels. Letter spacing is normal except tiny mono labels/stamps, where tracking is intentional.

Shape, Texture, Motion

  • Radii stay small: 2px, 4px, 6px. Pills are only for chips/avatars/status dots.
  • Cards and panels should feel cut, taped, stamped, pinned, written on, or stacked. No glossy rounded SaaS cards.
  • Shadows are warm paper lift, never dramatic glassmorphism.
  • Use paper grain, faint ruled notebook lines, pegboard dots, corkboard texture, dashed handoff rules, and graphite terminal surfaces.
  • Motion is calm and mechanical: 120ms, 200ms, 360ms. No bounce/spring personality.

Component Vocabulary

The approved bundle decomposes into these canonical components. Future builds should reuse these concepts and names where practical:

  • Core: Button, Panel, Stamp, Tag.
  • Agent identity: AgentAvatar.
  • Modules: AgentCard, AgentQueue, BuildReceipt, FrontRoomCard, HandoffCard, HumanNote, SystemDiagram, TraceLog.
  • Notes: NotebookNote, StickyNote.
  • Website kits: Nav, Hero, BackWall, FrontRoom, OpenNotebook, TeamRoster, WhoWeAre, SiteFooter.

Layout Language

The website should read as a live workspace, not a landing page pretending to be one. Build sections as rooms/surfaces:

  • Hero: brand/mark visible immediately, editorial confidence, work-surface texture.
  • Back wall: pegboard/system-board metaphor for persistent structure.
  • Front room: cards, notes, receipts, and traces that show the company operating.
  • Open notebook: explanatory copy and proof points on ruled paper.
  • Team roster: agents as a working team, not abstract mascots.
  • Who We Are: keep the approved copy direction separate from this visual standard.

Copy Boundary

The visuals in the approved source are binding. The copy is mostly placeholder. Existing approved copy and positioning still govern messaging, especially the locked Who We Are line and subhead in POSITIONING-BRIEF.md / project memory.

Supersession

This standard supersedes the earlier Claude design-system direction where there is conflict, especially around the mark. The previous note that the system needed to be less visually polite still applies; this approved file is the answer to that critique.
