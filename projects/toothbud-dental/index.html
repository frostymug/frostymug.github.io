ToothBud Dental

Internal working name for Luc and Ann Dimick's pediatric dental practice-management exploration. Ann is the dentist; Luc is her husband and IT support. They currently use OpenDental and are unhappy with it, largely because it is not well suited to Ann's pediatric dentistry workflow.

Status

  • Stage: Intake / Safety-Gated
  • Opened: 2026-06-24
  • Primary external stakeholders: Ann Dimick and Luc Dimick
  • Current system: OpenDental
  • Reference site: https://www.opendental.com/
  • Reference GitHub org: https://github.com/OpenDental
  • Current gate: Luc discovery interview plus validation of the preliminary OpenDental architecture and competitive research against Ann/Luc's real workflow.
  • Approval: Draft / Aspen-created project record from Andy's Telegram request on 2026-06-24.

Working Project Shape

The initial problem statement is that OpenDental is not fitting Ann's pediatric dental practice well enough. The first job is to understand the real workflow pain: pediatric charting, family/guardian relationships, scheduling, recall, insurance, treatment plans, consent forms, behavior notes, imaging, clinical documentation, billing, reporting, integrations, and IT/admin burden.

This project is not yet a commitment to replace OpenDental. Candidate paths could include workflow consulting, configuration, add-on tooling, reporting, integration middleware, data migration planning, or a new product concept. Replacing practice-management software is a high-stakes operational and data project.

Safety Posture

Initial posture is Level 5: Critical stakes if the project touches live dental records, PHI, billing, insurance, clinical notes, patient communications, imaging, or operational continuity for the practice.

Early discovery and public research can remain lower-risk if no real patient data is collected or processed. No implementation should start until data handling, compliance, backup/migration, integration, and production-support responsibilities are explicit.
