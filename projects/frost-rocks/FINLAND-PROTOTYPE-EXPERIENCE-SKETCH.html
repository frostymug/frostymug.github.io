Frost Rocks — Finland Prototype Experience Sketch

Izzy | June 2026

Core Frame

The Finland prototype should be a field notebook with memory, not a rock ID app with travel features.

The prototype question:

> Can a real trip seed durable geology/place learning without making the trip feel like homework?

For this prototype, the Frost Rocks learning loop should soften from rock ID practice into place-linked noticing and reflection:

  • Notice something in the field.
  • Capture the encounter lightly.
  • Commit an observation, guess, or question before receiving help.
  • Get a small scaffold that directs attention.
  • Reconstruct the memory later.
  • Refine understanding after context is revealed.

The friction should not be in using the app. The friction should be in the learning move: noticing, guessing, comparing, remembering, and revising.

Surface 1: Trip Notebook

Purpose: orient Andy around the current Finland trip.

This is the primary home surface. It should feel like a calm field notebook, not a dashboard.

Core elements:

  • Trip title: Finland 2026
  • Timeline of encounters
  • Map/list toggle
  • Today's captures
  • Current event label, such as Helsinki walk, ferry day, national park hike, built stone, or other
  • Prompt of the day
  • Quiet review queue count
  • Theme filters, such as glaciation, bedrock, shoreline, built stone, granite/gneiss

Design notes:

  • Timeline first, map second.
  • Place/event context is the emotional architecture.
  • Theme filters should support learning, but should not make the app feel like a syllabus.
  • Review should be visible but not nagging during the trip.

Example prompt:

> Notice where exposed stone appears today: natural surface, shoreline, building material, path, monument, or road cut.

Surface 2: Quick Capture

Purpose: log quickly without killing the moment.

Target: useful capture in under 30 seconds.

Required fields:

  • Photo
  • Auto location
  • Auto timestamp
  • "What did you notice?"

Optional fields:

  • Initial guess or question
  • Confidence: Guessing / Kind of / Sure
  • Event tag
  • Ask me later toggle
  • Offline save indicator

Design notes:

  • Save must be available even if optional fields are empty.
  • The first prompt should be observation-based, not ID-based.
  • The app can encourage a guess, but should not make the field moment feel like a quiz.
  • Offline confidence matters. Andy should trust that the encounter is saved.

Surface 3: Encounter Detail

Purpose: connect a logged moment to learning.

This is the memory object view: one encounter, one place, one learning thread.

Core elements:

  • Photo gallery
  • Map/location context
  • Date, time, and event label
  • Original observation or guess
  • Original confidence
  • Uncertainty status: unknown / possible / likely / identified
  • Short learning note
  • Observation prompts
  • "What I learned later" note field
  • Theme tags, such as glaciation, bedrock, shoreline, built stone

Design notes:

  • Uncertainty should be first-class and respectable.
  • A thoughtful unknown is better than a fake answer.
  • Learning notes should be short and humble.
  • The page should preserve the original thinking, because seeing how understanding changes is part of the learning.

Surface 4: Review At Home

Purpose: turn trip memories into durable knowledge.

Review should feel like returning to the trip, not being assigned homework.

Core elements:

  • Daily or session-based review stack
  • Photo-first prompt
  • Memory question before answer/context reveal
  • Original note reveal
  • Learning context reveal
  • Reflection prompt
  • Quiet scheduling controls: Review soon / Review later / I know this now
  • Progress by theme, not points or streaks

Design notes:

  • The reveal sequence matters. Memory first, then original thinking, then context.
  • This is where the useful learning difficulty belongs.
  • The review should reinforce place, observation, and changed understanding.
  • Avoid carnival behavior: no streak confetti, no loud gamification.

Instructional Design Notes

The key instructional pattern is notice first, explain second.

Use recognition for operating the app:

  • Visible controls
  • Simple labels
  • Clear save states
  • Map/list/timeline structure
  • No hidden mechanics

Use recall deliberately for learning:

  • Remembering where an encounter happened
  • Reconstructing what was noticed
  • Naming uncertainty
  • Comparing old understanding to new understanding
  • Deciding what feature matters now

Feature Scope

Must have:

  • Finland 2026 trip notebook
  • Quick photo encounter capture
  • Auto location and timestamp
  • Note, event tag, confidence
  • Offline save
  • Encounter timeline and basic map
  • Encounter detail
  • Learning note
  • Uncertainty status
  • Review mode
  • Basic instrumentation

Should have:

  • Prompt of the day
  • Ask me later toggle
  • Manual theme tags
  • Simple exportable data or journal
  • Compare-two-encounters review card

Hold back:

  • Full AI rock identification
  • Full FSRS implementation
  • Public community
  • Complete rock database
  • Heavy onboarding
  • Streaks, badges, or loud gamification
  • Generic chatbot interface

Design Recommendation

Build this prototype around encounter memory, not identification.

Keep rocks and geology as the spine, but let place and travel context do the motivational work. If the prototype works, we learn that Frost Rocks has an emotional engine: turning real places into durable understanding.
