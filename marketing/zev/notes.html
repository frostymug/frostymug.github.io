Zev Published HTML

Drop Zev-owned standalone HTML here when Andy wants it published on the gated docs site.

  • Source folder: /home/ubuntu/.openclaw/workspace/published-html/zev/
  • Published URL folder: https://docs.aspenfrost.com/zev/
  • Index page: https://docs.aspenfrost.com/zev/
  • Example: positioning-brief.html publishes to https://docs.aspenfrost.com/zev/positioning-brief.html

Rules:

  • Use static files only: HTML, CSS, images, fonts, and small JavaScript when genuinely needed.
  • Do not put secrets, tokens, private keys, or unpublished client-sensitive data here.
  • Keep filenames lowercase with hyphens, e.g. website-positioning-brief.html.
  • If a page needs assets, place them in a subfolder beside the HTML, e.g. assets/hero.png; the publisher preserves the folder tree.
  • Promotion to this folder means the page is review-ready for Andy, not necessarily externally approved.

Publishing:

The docs build copies this folder into /zev/ and generates the /zev/ index automatically. Manual publish is still:

cd /home/ubuntu/.openclaw/workspace/project-viewer
./scripts/publish.sh

Nightly docs publishing also picks up changes automatically.
